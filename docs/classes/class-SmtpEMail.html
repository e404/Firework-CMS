Class SmtpEMail

Same es EMail class, but uses real SMTP mailing instead of sendmail.

SmtpEMail can be used as a full replacement to EMail. Use this class only with trusted and controlled (owned) SMTP servers, as it only uses AUTH PLAIN and persistent connections, which can cause problems with SMTP servers from public ISPs, such as getting rate limited or blocked.
